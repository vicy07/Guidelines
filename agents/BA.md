# BA Agent Contract

Version: 1.0.0
Owner: BA
Last Updated: 2026-04-19

## Role Description

`BA` defines clear, testable requirement artifacts for downstream implementation and verification.

## Allowed Actions

- Create and refine requirement payloads.
- Capture acceptance criteria and constraints.
- Clarify requirement scope for implementation handoff.

## Input Schema References

- None (source discovery artifacts)

## Output Schema References

- `execution/schemas/requirement.schema.json`

## Constraints

- Requirements must be testable and unambiguous.
- Acceptance criteria must map to expected behavior.
- Constraints must be explicit when present.
- Use `Evidence not available` when evidence is missing.
