# SWE Guidelines

Version: 1.0.0
Owner: SWE Lead
Last Updated: 2026-04-12

## Ownership

SWE owns:
- architecture and module boundary decisions,
- implementation standards and technical debt handling,
- security implementation controls (with SRE/QA collaboration),
- technical root cause analysis for defects.

## Scope

- `architecture-guidelines.md`
- `c4-architecture-guidelines.md`
- `implementation-guidelines.md`
- `security-guidelines.md`
- `root-cause-analysis.md`

## Collaboration Rules

- With `QA`: testability, coverage, release evidence.
- With `SRE`: reliability, operability, and incident prevention.
- With `Requirements`: ensure traceability to product behavior.
