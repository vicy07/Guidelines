# Mission Format Standard

Version: 1.0.0
Owner: Repository Maintainer
Last Updated: 2026-04-12

## Metadata (required)

- Version
- Owner
- Last Updated
- Product Name
- Phase (MVP / Phase 2A / ...)

## Required Sections

1. Product Definition
2. Mission Statement
3. Practical Outcomes
4. Product Principles
5. Target Audience (current phase)
6. Non-goals (out of scope)
7. Success Metrics
8. Phase Evolution

## Writing Rules

- The mission should be 1-2 sentences.
- Principles should be written as decision-making rules.
- Non-goals are mandatory to remove scope ambiguity.
- Metrics must be measurable (number, percentage, time).

## Quality Checklist

- There is a clear boundary between goal and functionality.
- There are no contradictions with use cases.
- Success criteria are explicit.
