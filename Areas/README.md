# Areas

This folder groups the repository's normative standards by area rather than by top-level role folder.
The companion `Agents/` folder contains role profiles that point into these areas.

- `requirements/` - product and requirements standards owned by `BA` with `PO` co-ownership.
- `ux/` - UI/UX standards owned by `UX`.
- `swe/` - software engineering standards.
- `qa/` - quality assurance standards.
- `sre/` - site reliability and operations standards.

Use `guidelines-index.yaml` as the machine-readable source of truth for ownership, dependencies, and validation rules across these areas.
